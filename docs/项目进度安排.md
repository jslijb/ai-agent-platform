## 完整项目任务拆分（共 8 天）

基于我们讨论的 13 项 RAG 技术 + Multi-Agent + MCP + DeepWiki + 评估体系，重新拆分为每天可执行的、有明确验收标准的任务。

**前置条件**：Day 1 已完成（Next.js + Prisma + NextAuth + Docker + tRPC + 基础目录结构）。

### Day 2：LangGraph Agent 核心 + 工具调用基础

**目标**：让项目具备运行一个简单 Agent 的能力，并打通与阿里百炼模型的调用。

**任务清单**：
1. 安装依赖：`@langchain/langgraph`, `@langchain/core`, `langchain`, `zod`
2. 实现 `/app/server/llm/providers/bailian.ts`：封装阿里百炼 API 调用（兼容 OpenAI SDK）
3. 实现 `/app/server/agents/simpleAgent.ts`：一个 ReAct Agent，带 calculator 工具
4. 实现 `/app/api/agent/run/route.ts`：POST 接口，调用 Agent 并返回结果
5. 环境变量：`.env.local` 添加 `DASHSCOPE_API_KEY`

**验收标准**：
- [ ] `curl -X POST http://localhost:3000/api/agent/run -H "Content-Type: application/json" -d '{"query":"35除以7等于多少"}'` 返回正确计算结果
- [ ] Agent 的思考过程（调用 calculator 工具）能在终端日志中看到

---

### Day 3：向量检索 + 混合检索（BM25 + 向量 + RRF）

**目标**：实现工业级混合检索管道，支持文档上传、切片、双路召回、RRF 融合。

**任务清单**：
1. 安装依赖：`pgvector`, `@langchain/community`, `pdf-parse`, `nodejieba`（BM25 中文分词）
2. 数据库：在 PostgreSQL 中启用 `pgvector` 扩展，创建 `documents` 表和 `embeddings` 表
3. 实现 `/app/server/rag/chunking/semantic-chunker.ts`：使用 `SemanticChunker` 智能切片
4. 实现 `/app/server/rag/retrieval/hybrid-retriever.ts`：
   - 稠密检索：OpenAI embedding / 本地 embedding（Semnova）
   - 稀疏检索：BM25 实现
   - RRF 融合算法
5. 实现 `/app/api/document/upload/route.ts`：上传 PDF → 切片 → 生成 embedding → 存储
6. 实现 `/app/api/rag/search/route.ts`：接受查询，返回混合检索结果

**验收标准**：
- [ ] 上传一份财报 PDF，能正确切块并存入向量库
- [ ] 查询专有名词（如“NVDA”），BM25 能精确匹配，向量也能召回相关语义块
- [ ] RRF 融合后，Top-5 结果的相关性明显优于单一检索方式（可手动评估）

---

### Day 4：GraphRAG 知识图谱 + 多跳推理

**目标**：构建实体关系图谱，实现 Cypher 查询和多跳推理检索。

**任务清单**：
1. Neo4j 容器已启动（Day 1 已完成）
2. 安装依赖：`neo4j-driver`
3. 实现 `/app/server/rag/graph/entity-extractor.ts`：用 LLM 从文档中抽取 `(实体, 关系, 实体)` 三元组
4. 实现 `/app/server/rag/graph/graph-builder.ts`：批量写入 Neo4j
5. 实现 `/app/server/rag/graph/graph-retriever.ts`：根据问题中的实体，生成 Cypher 查询，执行多跳推理
6. 修改 `/app/api/graph/build/route.ts`：上传文档后自动构建图谱
7. 修改 `/app/api/rag/search/route.ts`：增加对 GraphRAG 结果的支持

**验收标准**：
- [ ] 上传包含“英伟达推出H100芯片，AMD 竞品 MI300”的文档，能抽取到 `(英伟达, 生产, H100)`、`(AMD, 竞争, 英伟达)` 等关系
- [ ] 查询“英伟达的竞争对手生产了什么芯片？”能返回 H100 或相关答案，且推理路径正确

---

### Day 5：RAG 高级优化（重排序 + 查询改写 + 父子文档）

**目标**：提升检索精度，减少噪音，增强答案完整性。

**任务清单**：
1. 安装依赖：`cross-encoder`, `@xenova/transformers`（BGE-reranker 本地运行）
2. 实现 `/app/server/rag/reranking/reranker.ts`：Cross-Encoder 对 Top-20 重排序输出 Top-3
3. 实现 `/app/server/rag/query/hyde-transformer.ts`：HyDE 查询改写（LLM 生成假设文档，再嵌入检索）
4. 实现父子文档合并：使用 `ParentDocumentRetriever`，小块检索，父块返回
5. 修改 `/app/api/rag/search/route.ts`：串联重排序 + 查询改写（可选开关）

**验收标准**：
- [ ] 重排序后，Top-3 结果准确率明显高于原 Top-3（准备 5 个测试用例）
- [ ] 输入模糊查询“芯片周期”，HyDE 改写后能召回“半导体景气度”相关文档
- [ ] 检索结果能自动补全父块内容，回答时上下文完整

---

### Day 6：多模态 RAG（图文混排 + 表格提取 + 答案溯源）

**目标**：让 Agent 能看懂 PDF 中的图片、表格，并生成带引用的答案。

**任务清单**：
1. 安装依赖：`llamaindex`, `@llamaindex/liteparse`, `llamaindex-ts`（注意 TypeScript 适配）
2. 实现 `/app/server/rag/multimodal/pdf-parser.ts`：用 LiteParse 解析图文混排 PDF，提取文本和图片位置
3. 实现 `/app/server/rag/multimodal/table-extractor.ts`：识别表格并转换为 Markdown 或 JSON
4. 实现 `/app/server/rag/citation/source-tracker.ts`：为每个 chunk 存储 `doc_id, page_num`
5. 实现 `/app/server/rag/citation/citation-injector.ts`：Prompt 要求 LLM 在答案中内嵌 `[来源: 文档, 页码]`
6. 修改 `/app/api/rag/search/route.ts`：返回结果包含溯源元数据
7. 修改 Agent 调用 RAG 工具时，要求输出带引用的答案

**验收标准**：
- [ ] 上传一份带图表和表格的财报 PDF，检索“营收增长趋势”，能返回表格数据而非“无法解析”
- [ ] 答案中每一条关键数据都带有 `[来源: XXX, 第X页]` 标记
- [ ] 点击引用（或前端展示）能定位到原始文档和页码

---

### Day 7：流式 RAG（增量索引） + Agentic RAG 自适应检索

**目标**：实现文档实时更新索引，Agent 能自主决定是否需要多次检索。

**任务清单**：
1. 安装依赖：`pg-ears`（PostgreSQL CDC 监听）或使用 Prisma 的 `@prisma/client` 监听 `updatedAt`
2. 实现 `/app/server/rag/streaming/cdc-listener.ts`：监听 `documents` 表的变更
3. 实现 `/app/server/rag/streaming/incremental-embedder.ts`：仅对变更文档重新切片、嵌入、更新向量库
4. 修改 Agent 的 LangGraph 状态机：增加“反思”节点，在生成答案前评估是否信息不足，若不足则触发新一轮检索（自适应检索）
5. 实现 `/app/server/agents/reflection-node.ts`：判断是否需要再检索
6. 修改 `/app/api/agent/run/route.ts`：支持 Agent 多轮调用 RAG 工具

**验收标准**：
- [ ] 更新某个文档内容后，无需手动重建索引，几秒内新内容可被检索到
- [ ] 提问“英伟达最新营收是多少？”如果第一次检索没找到，Agent 会自动改写查询并再次检索，最终给出答案
- [ ] 能在日志中看到 Agent 的迭代检索过程

4. 实现 `/scripts/run-evaluation.ts`：一键运行评估，输出报告
